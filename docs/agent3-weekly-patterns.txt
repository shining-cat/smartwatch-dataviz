Defensive Coding Features
--------------------------
- Minimum data threshold: 28 nights (4 weeks) for meaningful analysis
- Handles sparse data (not all weekdays may be present)
- Filters out invalid/null records before aggregation
- Shows "Need at least 4 weeks" message when insufficient data
- Console.log pattern detection results for debugging
- Safe handling of missing weekdays (returns null, skipped in chart)
- Graceful degradation with notice messages

Pattern Detection Logic
-----------------------
- Calculates weekly mean across all valid weekdays
- Computes deviation from mean for each day
- Flags significant deviations: >30 minutes OR >10% of mean
- Generates human-readable insights (e.g., "Mondays: -45 min below weekly avg")
- Weekend effect calculated separately (Sat-Sun vs Mon-Fri)
- Sorts patterns by absolute deviation strength
- Returns top 3 patterns plus weekend effect if significant

Data Model Compliance
---------------------
Follows SLEEP DATA MODEL CONTRACT:
- date: ISO date string (wake-up date)
- durationMinutes: Total sleep in minutes
- durationHours: Duration in hours (for display)
- dayOfWeek: 0=Sunday, 1=Monday, ..., 6=Saturday
- weekdayName: Human-readable day name
- bedtimeMin: Optional bedtime in minutes (from midnight)
- wakeMin: Optional wake time in minutes (from midnight)
